[general tags]: # "cli"

The Leo CLI is a command line interface tool that comes equipped with the Leo compiler.

:::tip
You can print the list of commands by running `leo --help`
:::

## Commands

- [`abi`](./abi.mdx) - Generate an ABI from a compiled `.aleo` bytecode file.
- [`account`](./account.mdx) - Create a new Aleo account, sign and verify messages.
  - [`new`](./account.mdx#leo-account-new) - Generates a new Aleo account.
  - [`import`](./account.mdx#leo-account-import) - Derive and Aleo account from a private key.
  - [`sign`](./account.mdx#leo-account-sign) - Sign a message using your Aleo private key.
  - [`verify`](./account.mdx#leo-account-verify) - Verify a message and signature from an Aleo address.
- [`add`](./add.mdx) - Add a new onchain or local dependency to the current project.
- [`build`](./build.mdx) - Compile the current project.
- [`clean`](./clean.mdx) - Clean the build and output artifacts.
- [`debug`](./debug.mdx) - Run the interactive debugger in the current package.
- [`deploy`](./deploy.mdx) - Deploy a program to the Aleo network.
- [`devnet`](./devnet.mdx) - Initialize a local devnet.
- [`devnode`](./devnode.mdx) - Start a lightweight development node that bypasses consensus and proof generation.
  - [`start`](./devnode.mdx#leo-devnode-start) - Start the devnode.
  - [`advance`](./devnode.mdx#leo-devnode-advance) - Manually advance the ledger by creating new blocks.
- [`execute`](./execute.mdx) - Execute a program and produce a transaction containing a proof.
- [`fmt`](./fmt.mdx) - Format Leo source files.
- [`new`](./new.mdx) - Create a new Leo project in a new directory.
- [`query`](./query.mdx) - Query live data and state from the Aleo network.
  - [`block`](./query.mdx#leo-query-block) - Query block information.
  - [`transaction`](./query.mdx#leo-query-transaction) - Query transaction information.
  - [`program`](./query.mdx#leo-query-program) - Query program source code and live mapping values.
  - [`stateroot`](./query.mdx#leo-query-stateroot) - Query the latest stateroot.
  - [`committee`](./query.mdx#leo-query-committee) - Query the current committee.
  - [`mempool`](./query.mdx#leo-query-mempool) - Query transactions and transmissions from the memory pool.
  - [`peers`](./query.mdx#leo-query-peers) - Query peer information.
- [`remove`](./remove.mdx) - Remove a dependency from the current project.
- [`run`](./run.mdx) - Run a program without producing a proof.
- [`test`](./test.mdx) - Run the test cases for a Leo project.
- [`update`](./update.mdx) - Update to the latest version of Leo.
- [`upgrade`](./upgrade.mdx) - Upgrade a deployed program on the Aleo network.
- [`synthesize`](./synthesize.mdx) - Generate proving and verifying keys for a program.

## Universal Flags

These flags are available to use alongside all commands in the Leo CLI.

#### `--help`

#### `-h`

Prints available commands and flags.

#### `--version`

#### `-V`

Prints the currently installed version of Leo.

#### `-q`

Suppresses the CLI output.

#### `-d`

Prints out additional information for debugging if possible.

#### `--path <PATH>`

Specifies the path to Leo program root folder. Defaults to `./`.

#### `--home <HOME>`

Specifies the path to the `.aleo` program registry. This is where programs downloaded from the network will be cached. Defaults to `~/.aleo/registry`.

#### `--json-output[=<PATH>]`

Saves structured JSON output to disk.

- **Default location**: `build/json-outputs/<command>.json`
- **Custom path**: `--json-output=my-results.json`

Supported commands: `deploy`, `upgrade`, `run`, `execute`, `test`, `query`, `synthesize`.

```bash title="Examples"
# Save to default location (build/json-outputs/run.json)
leo run --json-output main 1u32 2u32

# Save to custom path
leo execute main --json-output=my-results.json
```
